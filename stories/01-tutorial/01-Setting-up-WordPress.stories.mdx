# Setting up WordPress
In this section, we'll setup a new WordPress installation and theme to begin working on your Irving application.

## Prerequisites
You should have a working copy of WordPress.

[How to Install WordPress](https://wordpress.org/support/article/how-to-install-wordpress/).

## Install and activate the WP Irving plugin
Begin by downloading, installing, and activating the [WP Irving](https://github.com/alleyinteractive/wp-irving) plugin.

* You should see a new "Irving Development" dropdown in your [admin bar](http://localhost:6006/?path=/docs/packages-wordpress--page#admin-bar).
* Pretty permainks will need to be enabled,
  * You can confrim this by visiting `yourdomain.com/wp-json/irving/v1`.

## Setup your theme
One of the ways WP Irving helps streamine development is by offering a basic templating system, mimicing the [WordPress Template Hierarchy](https://developer.wordpress.org/themes/basics/template-hierarchy/).

By default, the [Components Endpoint](?path=/docs/components-components-endpoint--page) will be constructed for any given request using `.json` files in `/wp-content/your-theme/templates/` and `/wp-content/your-theme/template-parts/`.

We recommend this initial scaffolding,

```
wp-content
│
└───your-theme
    │   functions.php
    │   index.php
    │   style.css
    │
    └───templates
    │       archive.json
    │       author.json
    │       defaults.json
    │       frontpage.json
    │       index.json
    │       page.json
    │       single.json
    │
    └───template-parts
            header.json
            footer.json
```

[See Irving Twenty Twenty-One example](https://github.com/alleyinteractive/irving-twentytwentyone/tree/main/templates).

## Create `templates/defaults.json`
The `defaults.json` template is specific to Irving, and is used to populate the [`defaults` key in the Components Endpoint](?path=/docs/components-components-endpoint--page#defaults).

These components serve as the global layout, defining various parts of your site, only to be swapped for specific routes.

### Breaking it down
This is a simple `defaults.json` endpoint, with a header, body, and footer wrapper.

Both `irving/header-wrapper` and `irving/footer-wrapper` contain calls to `irving/template-part`, which loads the right partial, ex. `template-parts/header.json`.

### Template
```json

{
  "defaults": [
    {
      "name": "irving/header-wrapper",
      "children": [
        {
          "name": "irving/template-part",
          "config": {
            "slug": "site-header"
          }
        }
      ]
    },
    {
      "name": "irving/body-wrapper"
    },
    {
      "name": "irving/footer-wrapper",
      "children": [
        {
          "name": "irving/template-part",
          "config": {
            "slug": "site-footer"
          }
        }
      ]
    }
  ]
}
```

## Create `templates/index.json`
This template is the equivlent of `index.php` in a normal WordPress theme. It is at the furthest end of the [Template Hierarchy](https://developer.wordpress.org/themes/basics/template-hierarchy/).

### Breaking it down
This example `index.json` will display "Hello World" inside `irving/body-wrapper`, which was defined in `defaults.json`.

### Template
```json
{
  "page": [
    {
      "name": "irving/body-wrapper",
      "children": [
        {
          "name": "irving/text",
          "config": {
            "content": "Hello World"
          }
        }
      ]
    }
  ]
}
```

## Connect to the Node application
Now that you have a working Irving endpoint, open your `.env` and update the `API_ROOT_URL` to your WordPress instance.

You can confirm the "Hello World" by running `npm run dev` and navigating to the root `http://localhost:3001`.
